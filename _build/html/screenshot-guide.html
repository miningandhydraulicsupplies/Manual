Screenshot Guide for Documentation

This guide identifies where screenshots should be added to enhance the user documentation. Screenshots should be clear, high-resolution images showing the actual application interface.

General Guidelines

  • Resolution: Minimum 1920x1080 for desktop views

  • Format: PNG for UI screenshots, JPG for photos

  • Annotations: Use red boxes or arrows to highlight key areas

  • Consistency: Use same browser and zoom level for all screenshots

  • Privacy: Blur or redact sensitive customer/business data

Screenshot Naming Convention

Use this format: [module]-[feature]-[view].png

Examples:

  • order-board-main.png

  • calibration-add-equipment.png

  • admin-users.png

  • label-pickslip-form.png

Screenshot Storage

Store screenshots in: docs/images/

Reference in markdown: ![Description](images/filename.png)

Annotation Tools

Recommended tools for annotations:

  • Snagit - Professional screenshot tool

  • Greenshot - Free, open-source

  • ShareX - Free with annotation features

  • Windows Snipping Tool - Built-in Windows tool

Screenshot Checklist

Before adding screenshot:

  • [ ] Remove sensitive data

  • [ ] Ensure good resolution

  • [ ] Add annotations if needed

  • [ ] Use consistent styling

  • [ ] Optimize file size

  • [ ] Add descriptive alt text

  • [ ] Test image displays correctly
